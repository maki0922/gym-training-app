# ステップ6 機能仕様書: 種目マスタ管理

## 概要

トレーナー・オーナーが種目マスタを管理する画面。全ロールが閲覧・追加可能。編集・無効化はオーナーのみ。

---

## 画面一覧

| 画面 | URL | 対象ロール |
|------|-----|-----------|
| 種目マスタ管理 | `/admin/exercises` | 全員（トレーナー・オーナー） |

---

## 6-1. 種目一覧画面

### 表示対象

- デフォルト: `is_active = true` の種目のみ表示
- オーナーは「無効化済みを表示」トグルで `is_active = false` の種目も表示可能

### 表示項目

| 項目 | 内容 |
|------|------|
| 種目名 | `name` |
| カテゴリ | `category` |
| システム種目バッジ | `is_system: true` の種目に「システム」バッジを表示 |
| 無効バッジ | `is_active: false` の種目に「無効」バッジを表示（オーナーが無効表示ON時） |
| 編集ボタン | オーナーのみ表示 |
| 無効化/再有効化ボタン | オーナーのみ表示 |

### カテゴリフィルター

画面上部にカテゴリボタンを横並びで表示（スクロール可）。

| ボタン | 動作 |
|--------|------|
| すべて | 全カテゴリを表示（デフォルト） |
| 胸 | 胸カテゴリのみ表示 |
| 背中 | 背中カテゴリのみ表示 |
| 脚 | 脚カテゴリのみ表示 |
| 肩 | 肩カテゴリのみ表示 |
| 腕 | 腕カテゴリのみ表示 |
| 腹 | 腹カテゴリのみ表示 |
| 有酸素 | 有酸素カテゴリのみ表示 |
| その他 | その他カテゴリのみ表示 |

### レイアウト

- 一覧上部に「種目を追加」ボタン（右寄せ）
- カテゴリフィルターボタンの下に一覧を表示
- PC: テーブル形式 / モバイル: カード形式（ステップ5と同様）
- 各カテゴリ内は登録日時の昇順で表示

---

## 6-2. 種目追加機能（全ロールが実行可能）

### 追加ダイアログ

「種目を追加」ボタンをクリックするとダイアログを表示。

#### 入力フォーム

| フィールド | 種別 | バリデーション |
|-----------|------|--------------|
| 種目名 | text | 必須、1〜50文字 |
| カテゴリ | select | 必須、8カテゴリから選択 |

#### 送信時の処理（Server Action）

1. バリデーション（zod）
2. 同名・同カテゴリの種目が既に存在しないか確認（`is_active` 問わず）
   - 存在する場合: エラー「同じカテゴリに同名の種目が既に存在します」
3. `exercise_master` に挿入（`is_system: false`, `is_active: true`）
4. 成功時: ダイアログを閉じ、一覧を再取得して表示を更新

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 種目名未入力 | 「種目名を入力してください」 |
| 種目名が50文字超 | 「種目名は50文字以内で入力してください」 |
| カテゴリ未選択 | 「カテゴリを選択してください」 |
| 同名・同カテゴリが存在 | 「同じカテゴリに同名の種目が既に存在します」 |
| その他エラー | 「種目の追加に失敗しました。しばらく時間をおいて再試行してください」 |

## 6-3. 種目編集機能（オーナー権限のみ）

### 編集ダイアログ

一覧の「編集」ボタンをクリックするとダイアログを表示。現在の値をフォームの初期値として表示。

#### 編集可能フィールド

| フィールド | 種別 | バリデーション | 備考 |
|-----------|------|--------------|------|
| 種目名 | text | 必須、1〜50文字 | |
| カテゴリ | select | 必須 | |

#### 送信時の処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. バリデーション（zod）
3. 自分自身を除いて同名・同カテゴリの種目が存在しないか確認
4. `exercise_master` の `name`・`category` を更新
5. 成功時: ダイアログを閉じ、一覧を再取得して表示を更新

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 種目名未入力 | 「種目名を入力してください」 |
| 種目名が50文字超 | 「種目名は50文字以内で入力してください」 |
| カテゴリ未選択 | 「カテゴリを選択してください」 |
| 同名・同カテゴリが存在 | 「同じカテゴリに同名の種目が既に存在します」 |
| その他エラー | 「更新に失敗しました。しばらく時間をおいて再試行してください」 |

## 6-4. 種目無効化・再有効化機能（オーナー権限のみ）

### 無効化

一覧の「無効化」ボタンをクリックすると確認ダイアログを表示。

| 項目 | 内容 |
|------|------|
| タイトル | 「種目を無効化しますか？」 |
| 本文 | 「{name} を無効化します。過去のトレーニング記録は残ります。無効化後は種目選択に表示されなくなります。」 |
| ボタン | 「キャンセル」「無効化する」 |

#### 処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. `is_active = false` に更新
3. 成功時: 一覧を再取得して表示を更新

### 再有効化

オーナーが「無効化済みを表示」トグルをONにすると、無効な種目に「再有効化」ボタンが表示される。確認ダイアログなしで即時再有効化。

#### 処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. `is_active = true` に更新
3. 成功時: 一覧を再取得して表示を更新

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 無効化失敗 | 「無効化に失敗しました。しばらく時間をおいて再試行してください」 |
| 再有効化失敗 | 「再有効化に失敗しました。しばらく時間をおいて再試行してください」 |

## 6-5. シードデータ投入

### 対象データ

`is_system: true` として以下を投入:

| カテゴリ | 種目 |
|---------|------|
| 胸 | ベンチプレス, ダンベルフライ, インクラインベンチプレス, チェストプレス（マシン）, ケーブルクロスオーバー |
| 背中 | デッドリフト, ラットプルダウン, シーテッドロウ, ベントオーバーロウ, 懸垂 |
| 脚 | スクワット, レッグプレス, レッグカール, レッグエクステンション, ブルガリアンスクワット |
| 肩 | ショルダープレス, サイドレイズ, フロントレイズ, リアレイズ, アーノルドプレス |
| 腕 | アームカール, トライセプスプレスダウン, ハンマーカール, フレンチプレス |
| 腹 | クランチ, プランク, レッグレイズ, アブローラー |
| 有酸素 | トレッドミル, エアロバイク, ローイングマシン |
| その他 | ストレッチ, フォームローラー |

### 投入方法

Supabase マイグレーションで `INSERT ... ON CONFLICT DO NOTHING` を使用し、冪等性を担保する。
