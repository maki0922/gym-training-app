# ステップ8 機能仕様書: トレーニングセッション記録

## 概要

トレーナー・オーナーがトレーニングセッションを作成し、種目・セット・メモをリアルタイムで記録する画面。データは操作のたびにDBへ即時保存される（自動保存方式）。

---

## 画面一覧

| 画面 | URL | 対象ロール |
|------|-----|-----------|
| セッション記録（編集） | `/sessions/[id]/edit` | 全員 |

※ 新規セッション作成は専用ページを設けず、顧客詳細画面のボタンからServer Actionでセッションを作成し、`/sessions/[id]/edit` にリダイレクトする。

---

## データ保存方式

- **自動保存（リアルタイム保存）**: 「保存」ボタンは存在しない
- 「新規セッション開始」時点でセッションをDBに作成（`status: in_progress`）
- 種目追加・セット追加・メモ入力など、各操作のたびにDBへ即時保存
- ブラウザを閉じてもデータは消えない
- 最後に「セッション完了」ボタンで `status: completed` に確定

### コンポーネント設計方針

セッション記録画面は `'use client'` コンポーネントとして実装する。

- **初回データ**: Server Component（ページ）でDBから取得し、propsとしてClient Componentに渡す
- **以降の操作**: ローカルステート（`useState`）で画面を管理し、バックグラウンドでServer Actionを呼んでDBに保存する
- **セット更新時に `revalidatePath` を呼ばない**: 入力中の画面リフレッシュによるフォーカス喪失を防ぐため。種目追加・セット追加・削除など構造的変更のみ `revalidatePath` を使用する

---

## 8-1. 新規セッション開始

### エントリーポイント

顧客詳細画面（`/customers/[id]`）に「新規セッション開始」ボタンを設置。

### 処理フロー（2段階）

**ステージ1: 重複チェック**

1. 「新規セッション開始」ボタンをクリック
2. Server Action `checkDuplicateSession` を呼び出し、同日・同顧客の既存セッションを検索
3. 既存セッションが見つからない場合 → ステージ2へ直接進む
4. 既存セッションが見つかった場合 → 重複警告ダイアログを表示（8-2参照）

**ステージ2: セッション作成**

1. Server Action `createSession` を呼び出す
2. `training_sessions` テーブルにレコード作成
   - `customer_id`: 選択した顧客のID
   - `trainer_id`: ログイン中のユーザーID
   - `session_date`: 今日の日付（JST）
   - `status`: `in_progress`
3. 作成したセッションIDで `/sessions/[id]/edit` にリダイレクト

---

## 8-2. 同日・同顧客の重複セッション警告

### 発動条件

同じ顧客に対して同日（session_date）に既に `in_progress` または `completed`（`is_deleted = false`）のセッションが存在する場合。

### 警告ダイアログの表示内容

| 項目 | 内容 |
|------|------|
| タイトル | 「本日のセッションが既に存在します」 |
| 本文 | 「担当: {トレーナー名} / ステータス: {入力中 or 完了}」（複数ある場合はリスト表示） |
| ボタン1 | 「既存のセッションを開く」→ 既存セッションの編集画面（`/sessions/[id]/edit`）に遷移。複数ある場合は最新のものを開く |
| ボタン2 | 「新規作成する」→ ステージ2を実行し、新規セッションを作成 |

---

## 顧客詳細画面への追加: 入力途中セッション通知

### 表示条件

この顧客に対して `status = 'in_progress'` かつ `is_deleted = false` のセッションが存在する場合。

### 表示内容

顧客詳細画面（`/customers/[id]`）の基本情報セクションと過去セッション一覧の間に通知バナーを表示。

| 項目 | 内容 |
|------|------|
| バナーテキスト | 「入力途中のセッションがあります（{session_date} / 担当: {トレーナー名}）」 |
| アクション | バナーをタップ → `/sessions/[id]/edit` に遷移 |
| スタイル | 黄色背景のアラートバナー（目立つ表示） |

## 8-3. セッション記録画面レイアウト

### ヘッダー部分

| 項目 | 内容 |
|------|------|
| 「← 戻る」リンク | `/customers/[customer_id]` に遷移 |
| 顧客名 | セッションに紐づく顧客の氏名 |
| セッション日付 | `YYYY年MM月DD日` 形式 |
| ステータス | `in_progress` の場合のみ「入力中」バッジを表示 |

### メイン部分

- 追加済みの種目をカード形式で縦に並べる
- 各種目カードの中にセットの入力行を表示
- 画面下部に「種目を追加」ボタン

### フッター部分

- セッション全体のメモ入力エリア
- 「セッション完了」ボタン（種目が1つ以上あるときのみ有効）

---

## 8-3 & 8-4. 種目追加: 2段階選択

### ステップ1: 部位カテゴリ選択（ダイアログ）

8カテゴリ + 自由入力をボタングリッドで表示。

| カテゴリ |
|---------|
| 胸 / 背中 / 脚 / 肩 / 腕 / 腹 / 有酸素 / その他 |
| 自由入力 |

### ステップ2: 種目一覧から選択

選択したカテゴリに属する `is_active = true` の種目をリスト表示。

- 種目名をタップすると種目を追加してダイアログを閉じる

### 処理（Server Action）

1. `training_records` テーブルに挿入
   - `session_id`: 現在のセッションID
   - `exercise_id`: 選択した種目のID
   - `sort_order`: 現在の種目数 + 1
2. **初回セットを自動作成**: 同時に `training_sets` に空のセットを1行挿入（`set_number: 1`, `weight: NULL`, `reps: NULL`）
3. ページを再取得（`revalidatePath`）

> **初回セット自動作成の理由**: 種目追加後にさらに「セットを追加」をタップする手間を省き、タップ数を最小限にするため。

---

## 8-5. 種目追加: 自由入力

### 自由入力フォーム

カテゴリ選択画面の「自由入力」ボタンからテキスト入力フィールドを表示。

| フィールド | 種別 | バリデーション |
|-----------|------|--------------|
| 種目名 | text | 必須、1〜50文字 |

### 処理（Server Action）

1. `training_records` テーブルに挿入
   - `exercise_id`: NULL
   - `exercise_name_manual`: 入力した種目名
   - `sort_order`: 現在の種目数 + 1
2. **初回セットを自動作成**: 同時に `training_sets` に空のセットを1行挿入

### 前回記録との関係

自由入力種目（`exercise_id = NULL`）は前回記録の検索対象外とする。`exercise_name_manual` による文字列一致は表記ゆれによる誤マッチングの恐れがあるため。

---

## 8-6. セット記録入力

### セット入力行の構成

各種目カード内に、セットごとの入力行を表示。

| フィールド | 種別 | 単位 | 必須/任意 | バリデーション |
|-----------|------|------|----------|--------------|
| 重量 | number | kg | 任意 | 0〜999.9、小数点第1位まで |
| 回数 | number | 回 | 任意 | 0〜999、整数のみ |
| メモ | text | — | 任意 | 100文字以内 |

- 数値フィールドは `inputmode="decimal"`（重量）/ `inputmode="numeric"`（回数）を使用
- タップターゲットは44px以上
- セット番号を行の左端に表示（「1」「2」「3」...）

### セット追加ボタン

「＋ セットを追加」ボタンをタップするとセット入力行を1行追加し、DBに保存。

### 処理（Server Action: セット追加）

1. `training_sets` テーブルに挿入
   - `record_id`: 対象 training_record のID
   - `set_number`: 現在のセット数 + 1
   - `weight`: NULL（初期値）
   - `reps`: NULL（初期値）
2. ページを再取得（`revalidatePath`）

### 処理（Server Action: セット更新）

入力フィールドからフォーカスが外れた（`onBlur`）タイミングでDBに保存。

1. ローカルステートを即座に更新（ユーザーには即座に反映して見える）
2. バックグラウンドでServer Actionを呼び出し、`training_sets` テーブルの該当レコードを更新
   - `weight`, `reps`, `notes` を更新
3. `revalidatePath` は呼ばない（フォーカス喪失防止）
4. 保存失敗時はトーストでエラーを表示

---

## 8-7 & 8-8. 前回記録の表示と「前回をコピー」

### 表示条件

- 同じ顧客の過去セッション（`status: completed`、`is_deleted = false`）に同じ `exercise_id` の記録が存在する場合
- **自由入力種目（`exercise_id = NULL`）は対象外**

### 表示内容（種目カード内）

「前回: {YYYY/MM/DD} — {セット数}セット」

- 前回の個別セットの重量・回数は表示領域を圧迫するため、セット数のみの簡易表示とする
- 詳細は「前回をコピー」で確認できる

### 「前回をコピー」ボタン

- タップすると前回のセット数・重量・回数・セットメモを現在の入力フィールドにコピー
- **処理手順**:
  1. 現在の種目に紐づく既存のセットをすべてDB上から削除
  2. 前回のセットと同じ数のセットを新規作成し、重量・回数・メモをコピー
  3. ローカルステートを更新し画面に反映
- コピー後、トレーナーは変更したい箇所だけ修正すればよい
- 確認ダイアログなし（すぐコピーしたい操作のため。誤操作時はセットを手動修正可能）

---

## 8-9. 自動保存の実装

### 保存タイミング

| 操作 | 保存タイミング | revalidatePath |
|------|--------------|---------------|
| 種目追加 | 追加直後（Server Action） | する |
| セット追加 | 追加直後（Server Action） | する |
| 重量・回数・セットメモの変更 | `onBlur`（フォーカスが外れたとき） | しない |
| 種目ごとのメモ変更 | `onBlur` | しない |
| セッション全体メモの変更 | `onBlur` | しない |
| 種目の削除 | 確認後（Server Action） | する |
| セットの削除 | 確認後（Server Action） | する |
| 前回をコピー | タップ直後（Server Action） | する |

---

## 8-10. 種目・セットの削除

### セットの削除

- 各セット行の右端に削除ボタン（ゴミ箱アイコン）
- 確認ダイアログを表示

| 項目 | 内容 |
|------|------|
| タイトル | 「セットを削除しますか？」 |
| 本文 | 「このセットの記録を削除します。」 |
| ボタン | 「キャンセル」「削除する」 |

- `training_sets` テーブルから該当レコードを削除
- 削除後、同じ種目の残りセットの `set_number` を1から振り直す

### 種目の削除

- 各種目カードの右上に削除ボタン（ゴミ箱アイコン）
- 確認ダイアログを表示

| 項目 | 内容 |
|------|------|
| タイトル | 「種目を削除しますか？」 |
| 本文 | 「{種目名} とそのセット記録をすべて削除します。」 |
| ボタン | 「キャンセル」「削除する」 |

- `training_records` テーブルから該当レコードを削除（関連する `training_sets` はDBのカスケード削除設定により自動削除される ※設定確認済み）
- 削除後、残りの種目の `sort_order` を1から振り直す

|-----------|--------------|

## 8-13. 「セッション完了」ボタン

### 表示条件

- 種目が1つ以上追加されている場合のみボタンを有効化

### 確認ダイアログ

| 項目 | 内容 |
|------|------|
| タイトル | 「セッションを完了しますか？」 |
| 本文 | 「完了後もセッション詳細画面から編集できます。」 |
| ボタン | 「キャンセル」「完了する」 |

### 処理（Server Action）

1. `training_sessions.status` を `completed` に更新
2. `/customers/[customer_id]` にリダイレクト
