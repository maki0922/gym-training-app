# ステップ10 機能仕様書: ダッシュボード

## 概要

ログイン後のホーム画面。本日のセッション状況と入力途中セッションの通知、直近の記録へのクイックアクセスを提供する。

コンテンツ領域の最大幅は `max-w-4xl` に制限し、大画面での視認性を確保する。

---

## 画面

| 画面 | URL | 対象ロール |
|------|-----|-----------|
| ダッシュボード | `/` | 全員（トレーナー・オーナー） |

---

## 10-1. 本日のセッション一覧

### 表示内容

本日（JST）の `training_sessions`（`is_deleted = false`）を一覧表示。

| 項目 | 内容 |
|------|------|
| 顧客名 | `customers.name` |
| ステータス | `in_progress`: 「入力中」バッジ / `completed`: 「完了」バッジ |
| 担当トレーナー | `profiles.display_name` |
| 種目数 | セッション内の `training_records` の件数 |

### レイアウト

- セクションタイトル: 「本日のセッション」
- コンパクトリスト形式（1つのCardコンポーネント内で区切り線で分割）
- セッションがない場合: 「本日のセッションはまだありません」
- 各行をタップ → `/sessions/[id]`（完了セッション） / `/sessions/[id]/edit`（入力中セッション）に遷移

---

## 10-2. 入力途中セッションの通知

### 表示条件

全顧客にわたり `status = 'in_progress'` かつ `is_deleted = false` のセッションが存在する場合（本日以外も含む）。

### 表示内容

ダッシュボード上部にアラートバナーを表示。

| 項目 | 内容 |
|------|------|
| バナーテキスト | 「入力途中のセッションが {N} 件あります」 |
| 各セッション情報 | 「{顧客名} — {session_date}（担当: {トレーナー名}）」 |
| アクション | 各セッションをタップ → `/sessions/[id]/edit` に遷移 |
| スタイル | 黄色背景のアラートバナー |

---

## 10-3. 直近の記録へのクイックアクセス

### 表示内容

直近7日間の `completed` セッション（`is_deleted = false`）を最大5件表示。

| 項目 | 内容 |
|------|------|
| 顧客名 | `customers.name` |
| 日付 | `session_date`（`YYYY/MM/DD` 形式） |
| 種目数 | セッション内の `training_records` の件数 |

### レイアウト

- セクションタイトル: 「直近の記録」
- コンパクトリスト形式（1つのCardコンポーネント内で区切り線で分割）
- セッションがない場合: 「直近の記録はありません」
- 各行をタップ → `/sessions/[id]` に遷移

---

## 権限

ダッシュボードは全ロール共通。表示データも共通（ジム内の全セッションが見える）。
