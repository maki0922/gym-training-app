# ステップ7 機能仕様書: 顧客管理

## 概要

トレーナー・オーナーが顧客情報を管理する画面。全ロールが一覧閲覧・追加・編集可能。無効化（退会処理）はオーナーのみ。

---

## 画面一覧

| 画面 | URL | 対象ロール |
|------|-----|-----------|
| 顧客一覧 | `/customers` | 全員（トレーナー・オーナー） |
| 顧客詳細 | `/customers/[id]` | 全員（トレーナー・オーナー） |

---

## 7-1. 顧客一覧画面

### 表示対象

- デフォルト: `is_active = true` の顧客のみ表示
- オーナーは「退会済みを表示」トグルで `is_active = false` の顧客も表示可能

### 表示項目

| 項目 | 内容 |
|------|------|
| 氏名 | `name` |
| フリガナ | `name_kana` |
| 性別 | `gender`（男性 / 女性 / その他）。未設定の場合は非表示 |
| 退会バッジ | `is_active: false` の顧客に「退会済み」バッジを表示（オーナーがトグルON時） |
| 編集ボタン | 全ロールに表示（`is_active = true` の顧客のみ） |
| 無効化ボタン | オーナーのみ表示（`is_active = true` の顧客のみ） |
| 再有効化ボタン | オーナーのみ表示（`is_active = false` の顧客のみ、トグルON時） |

### 検索・フィルタリング

画面上部に検索フィールドを配置。

| 機能 | 動作 |
|------|------|
| 氏名検索 | `name` または `name_kana` に対する部分一致検索（前方一致ではなく部分一致） |

- 検索はクライアントサイドでフィルタリング（顧客数が少ない想定のため）
- 検索フィールドにはプレースホルダー「氏名・フリガナで検索」を表示

### レイアウト

- 一覧上部に「顧客を追加」ボタン（右寄せ）
- 検索フィールドの下に一覧を表示
- PC: テーブル形式 / モバイル: カード形式
- フリガナの五十音順（昇順）で表示
- 各顧客の行 / カードをクリックすると顧客詳細画面に遷移

## 7-2. 顧客追加機能（全ロールが実行可能）

### 追加ダイアログ

「顧客を追加」ボタンをクリックするとダイアログを表示。

#### 入力フォーム

| フィールド | 種別 | 必須/任意 | バリデーション |
|-----------|------|----------|--------------|
| 氏名 | text | 必須 | 1〜50文字 |
| フリガナ | text | 必須 | 1〜50文字、全角カタカナのみ |
| 性別 | select | 任意 | 男性 / 女性 / その他 から選択（未選択可） |
| 生年月日 | date | 任意 | 過去の日付のみ（未来日は不可） |
| メモ | textarea | 任意 | 500文字以内 |

#### 送信時の処理（Server Action）

1. バリデーション（zod）
2. `customers` テーブルに挿入（`is_active: true`）
3. 成功時: ダイアログを閉じ、一覧を再取得して表示を更新
4. 成功トースト: 「顧客を追加しました」

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 氏名未入力 | 「氏名を入力してください」 |
| 氏名が50文字超 | 「氏名は50文字以内で入力してください」 |
| フリガナ未入力 | 「フリガナを入力してください」 |
| フリガナが50文字超 | 「フリガナは50文字以内で入力してください」 |
| フリガナが全角カタカナ以外 | 「フリガナは全角カタカナで入力してください」 |
| 生年月日が未来日 | 「生年月日に未来の日付は指定できません」 |
| メモが500文字超 | 「メモは500文字以内で入力してください」 |
| その他エラー | 「顧客の追加に失敗しました。しばらく時間をおいて再試行してください」 |

## 7-3. 顧客編集機能（全ロールが実行可能）

### 編集ダイアログ

一覧の「編集」ボタンをクリックするとダイアログを表示。現在の値をフォームの初期値として表示。

#### 編集可能フィールド

| フィールド | 種別 | 必須/任意 | バリデーション | 備考 |
|-----------|------|----------|--------------|------|
| 氏名 | text | 必須 | 1〜50文字 | |
| フリガナ | text | 必須 | 1〜50文字、全角カタカナのみ | |
| 性別 | select | 任意 | 男性 / 女性 / その他 | 未選択に戻すことも可能 |
| 生年月日 | date | 任意 | 過去の日付のみ | クリア可能 |
| メモ | textarea | 任意 | 500文字以内 | |

#### 送信時の処理（Server Action）

1. バリデーション（zod）
2. `customers` テーブルの該当レコードを更新
3. 成功時: ダイアログを閉じ、一覧を再取得して表示を更新
4. 成功トースト: 「顧客情報を更新しました」

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 氏名未入力 | 「氏名を入力してください」 |
| 氏名が50文字超 | 「氏名は50文字以内で入力してください」 |
| フリガナ未入力 | 「フリガナを入力してください」 |
| フリガナが50文字超 | 「フリガナは50文字以内で入力してください」 |
| フリガナが全角カタカナ以外 | 「フリガナは全角カタカナで入力してください」 |
| 生年月日が未来日 | 「生年月日に未来の日付は指定できません」 |
| メモが500文字超 | 「メモは500文字以内で入力してください」 |
| その他エラー | 「更新に失敗しました。しばらく時間をおいて再試行してください」 |

## 7-4. 顧客無効化・再有効化機能（オーナー権限のみ）

### 無効化（退会処理）

一覧の「無効化」ボタンをクリックすると確認ダイアログを表示。

| 項目 | 内容 |
|------|------|
| タイトル | 「顧客を退会処理しますか？」 |
| 本文 | 「{name} さんを退会処理します。過去のトレーニング記録は残ります。退会後は顧客一覧に表示されなくなります。」 |
| ボタン | 「キャンセル」「退会処理する」（destructive スタイル） |

#### 処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. `is_active = false` に更新
3. 成功時: 一覧を再取得して表示を更新
4. 成功トースト: 「退会処理が完了しました」

### 再有効化

オーナーが「退会済みを表示」トグルをONにすると、退会済み顧客に「再有効化」ボタンが表示される。確認ダイアログなしで即時再有効化。

#### 処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. `is_active = true` に更新
3. 成功時: 一覧を再取得して表示を更新
4. 成功トースト: 「再有効化しました」

#### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 無効化失敗 | 「退会処理に失敗しました。しばらく時間をおいて再試行してください」 |
| 再有効化失敗 | 「再有効化に失敗しました。しばらく時間をおいて再試行してください」 |

## 7-5. 顧客詳細画面

### URL

`/customers/[id]`

### 表示内容

#### 基本情報セクション

| 項目 | 表示 |
|------|------|
| 氏名 | `name` |
| フリガナ | `name_kana` |
| 性別 | 男性 / 女性 / その他（未設定の場合は「未設定」） |
| 生年月日 | `YYYY年MM月DD日` 形式（未設定の場合は「未設定」） |
| 年齢 | 生年月日から自動計算して表示（未設定の場合は非表示） |
| メモ | `notes`（未入力の場合は非表示） |
| ステータス | `is_active = false` の場合に「退会済み」バッジを表示 |

- 右上に「編集」ボタンを表示（全ロール）→ クリックで編集ダイアログを表示
- オーナーのみ「退会処理」ボタンを表示（`is_active = true` の場合のみ）

#### 過去セッション一覧セクション

| 項目 | 表示 |
|------|------|
| セッション日付 | `session_date`（`YYYY年MM月DD日` 形式） |
| 担当トレーナー | `profiles.display_name`（退職済みの場合は「{名前}（退職済み）」） |
| ステータス | `in_progress`: 「入力中」バッジ / `completed`: 表示なし |
| 種目数 | セッションに含まれる種目の数 |

- 日付降順で表示
- 各セッション行をクリックするとセッション詳細画面（`/sessions/[id]`）に遷移
- `is_deleted = true` のセッションは表示しない
- セッションがない場合は「トレーニング記録はまだありません」を表示
- ページネーション: 最初は20件表示、「もっと見る」ボタンで20件ずつ追加読み込み

### レイアウト

- 上部に「← 顧客一覧に戻る」リンク
- 基本情報セクションと過去セッション一覧セクションを縦に配置
- モバイル: カード形式でセッション一覧を表示
- PC: テーブル形式でセッション一覧を表示
