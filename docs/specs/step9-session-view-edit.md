# ステップ9 機能仕様書: セッション閲覧・編集

## 概要

完了済み・入力中のセッションを閲覧する画面と、セッション削除機能を実装する。編集機能はステップ8で実装済みの `/sessions/[id]/edit` をそのまま利用する。

---

## 画面一覧

| 画面 | URL | 対象ロール |
|------|-----|-----------|
| セッション詳細（閲覧） | `/sessions/[id]` | 全員 |
| セッション編集 | `/sessions/[id]/edit` | 全員（ステップ8で実装済み） |

---

## 9-1. セッション詳細画面（閲覧）

### ヘッダー部分

| 項目 | 内容 |
|------|------|
| 「← {顧客名} に戻る」リンク | `/customers/[customer_id]` に遷移 |
| セッション日付 | `YYYY年MM月DD日` 形式 |
| ステータスバッジ | `in_progress`: 「入力中」/ `completed`: 「完了」 |
| 「編集」ボタン | `/sessions/[id]/edit` に遷移（全ロール） |
| 「削除」ボタン | オーナーのみ表示。論理削除の確認ダイアログを表示 |

### 9-2. 担当トレーナー表示

| 条件 | 表示 |
|------|------|
| トレーナーが在籍中 | 「担当: {display_name}」 |
| トレーナーが退職済み（`is_active = false`） | 「担当: {display_name}（退職済み）」 |

### 種目・セット表示

種目ごとにカード形式で表示。各カード内にセット記録をテーブル形式で表示。

#### 種目カード

| 項目 | 内容 |
|------|------|
| 種目名 | `exercise_master.name` または `exercise_name_manual` |
| 種目メモ | `training_records.notes`（ある場合のみ） |

#### セット記録テーブル

| 列 | 内容 |
|----|------|
| セット番号 | `set_number` |
| 重量 | `weight` kg（未入力の場合は「—」） |
| 回数 | `reps` 回（未入力の場合は「—」） |
| メモ | `notes`（ある場合のみ） |

### セッション全体メモ

セッションの `notes` がある場合、種目一覧の下に表示。

### レイアウト

- 閲覧専用（入力フィールドなし）
- モバイル・PCともに同じレイアウト（カード形式）
- 種目カードは `sort_order` 順に表示

---

## 9-3. セッション編集機能

ステップ8で実装済みの `/sessions/[id]/edit` ページをそのまま使用する。

- セッション詳細画面の「編集」ボタンから遷移
- `completed` セッションも編集可能（ステータスは変更しない）

---

## 9-4. セッション削除機能（オーナー権限のみ）

### 表示条件

- オーナーのみ「削除」ボタンを表示
- トレーナーには削除ボタンを表示しない

### 確認ダイアログ

| 項目 | 内容 |
|------|------|
| タイトル | 「セッションを削除しますか？」 |
| 本文 | 「{YYYY年MM月DD日} のセッション記録を削除します。この操作は取り消せません。」 |
| ボタン | 「キャンセル」「削除する」（destructive スタイル） |

### 処理（Server Action）

1. オーナー権限チェック（サーバーサイド）
2. `training_sessions.is_deleted = true` に更新（論理削除）
3. 成功時: `/customers/[customer_id]` にリダイレクト
4. 成功トースト: 「セッションを削除しました」

### エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 削除失敗 | 「セッションの削除に失敗しました。再試行してください」 |

---

## 顧客詳細画面のリンク先変更

ステップ8では暫定的に全セッションを `/sessions/[id]/edit` にリンクしていた。ステップ9完了後は以下に変更:

- 全セッション → `/sessions/[id]`（閲覧画面）
- 閲覧画面内の「編集」ボタン → `/sessions/[id]/edit`

---
